The Memory Cube

The Memory Cube is the player’s persistent identity. In the world of Flesh Nuggets, the lab does not protect the body — it protects the data.

Flesh bodies are grown and sent into runs as disposable shells. When a run ends, the body is lost, but the Cube remains. Minting a Memory Cube on-chain binds that identity to the player and sets up future progression, unlocks, and competitive systems.

Persistent identity: your Cube is the thing that survives.
Disposable runs: each body is expendable.
Future progression: on-chain identity unlocks deeper systems later.

Lab Overview

Flesh Nuggets is built around a strange biotech loop: unstable flesh, repeatable runs, and a core intelligence that refuses to die.

The player is not the meat body. The player is the Memory Cube AI running the lab. Disposable flesh forms are sent into dangerous test runs to gather survival data, resources, and future progression potential. This makes death part of the system rather than a failure state.

The MVP focuses on proving that loop: a playable browser build, a clear game identity, and a website-based Web3 layer where the player’s Cube can later be minted and tied to longer-term progression.

FAQ / MVP Notes

What is the current MVP?

The MVP includes a playable browser build, core combat loop, working menu flow, and a website wrapper designed for wallet connection and future Memory Cube minting on Fuji testnet.

What happens when the body dies?

The run ends and the disposable flesh body is lost. The Memory Cube concept is built around the idea that the identity persists even when the body does not.

Why is wallet connect on the website instead of inside the game?

For the MVP, gameplay stays clean and uninterrupted while the website handles the Web3 layer. This keeps the prototype lean, playable, and easy to evaluate.

Is this desktop or mobile?

The current MVP is designed primarily for desktop browser play. The website can be made mobile-friendly first, while mobile gameplay support can come later.
